Build the code

Download the latest code release from: GitHub releases page

At the moment there are not pre-built binaries available for MirIDE and MirComposer.

Java 17

You need Java 17 to build and run MirComp.
Java is available for all the major operating systems (Windows, Linux and OSX).
There are several excellent open source options out there.
For example you can get OpenJdk 17 from jdk.java.net.
Untar the downloaded file and move the content to a folder of your choice (e.g. programs).

JavaFX 17.0.0.2

Download JavaFx 17.0.0.2 SDK and JMODS from the official website.
Unzip both files and move their content to a folder of your choice.

Gradle 7.3

Finally you need Gradle to build MirComp.
While there are several ways to install gradle we describe the manual one.
Download the Gradle 7.3 binary-only zip from the official web site.
As always unzip the folder and move its content to a folder of your choice.

Set up your environment variables

Supposing the folder is named "Programs"
On Windows 10 you can open a powershell and enter:

[Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Programs\jdk-17.0.1", [System.EnvironmentVariableTarget]::Process)
[Environment]::SetEnvironmentVariable("JRE_HOME", $env:JAVA_HOME, [System.EnvironmentVariableTarget]::Process)
[Environment]::SetEnvironmentVariable("JAVAFX_HOME", "C:\Programs\javafx-sdk-17.0.2", [System.EnvironmentVariableTarget]::Process)
[Environment]::SetEnvironmentVariable("JAVAFX_JMODS_HOME", "C:\Programs\javafx-jmods-17.0.2", [System.EnvironmentVariableTarget]::Process)
[Environment]::SetEnvironmentVariable("GRADLE_HOME", "C:\Programs\gradle-7.3.3", [System.EnvironmentVariableTarget]::Process)
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";" + $env:JAVA_HOME + "\bin;" + $env:GRADLE_HOME + "\bin", [System.EnvironmentVariableTarget]::Process)

On Ubuntu 20.04:

export JAVA_HOME=/home/programs/jdk-17
export JRE_HOME=$JAVA_HOME
export JAVAFX_HOME=/home/programs/javafx-sdk-17.0.2
export JAVAFX_JMODS_HOME=/home/programs/javafx-jmods-17.0.2
export GRADLE_HOME=/home/programs/gradle-7.3
export PATH=$PATH:$JAVA_HOME/bin:$GRADLE_HOME/bin

Create a self-contained private MirIDE (or MirComposer) application

NOTE: MirComposer is a Lite version of MirIDE that does not support Groovy scripting and uses an experimental text editor without the javafx.web component. Moreover, it does not support projects but only single files.

We are almost ready to finally create our own desktop application.
First, create a dist/miride (or dist/mircomposer) folder and then launch the gradle createApp task
On Ubuntu 20.04:

mkdir dist
cd dist
mkdir miride
cd /home/programs/mircomp/miride
gradle createApp -PdistPath="/home/programs/dist/miride"
You can finally run the main executable inside the created self-contained application:
cd /home/programs/dist/miride/app/MirIDE/bin
./MirIDE
Similar on Windows 10: 
C:\Programs\mircomp\miride> gradle createApp -PdistPath="C:\Programs\dist\miride"
where inside the just created dist folder is sufficient a double click on the exec file.
